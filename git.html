GIT

Git is a distributed version control system which was made by Linus Torvalds. Git is a very popular software due to its flexibility and easy usage. Git provides us a platform where we can do anything with our code and still can fallback. Git is superuseful when it comes to bigger and real life projects. That is a reason why software engineers should use it. In this page, we have listed the git commands which you will be using the most at the time of software development. However, git is not limited to this, it can go far beyond and that is something more advanced and hence, out of the scope of this page.

Git Official Website

Initial Steps

The following commands are single use and are used to configure git with your username and email

Set Username

$ git config --global user.name "User Name"
Configure your git username as User Name with this command. Your username appears when you commit changes in your repository

Set Email

$ git config --global user.email "user@xyz.com"
Configure your email as user@xyz.com with this command. Your email will be stored when you commit changes in your repository.

Using git commands in your repository

Initialize Git Repository

$ git init
Initialize empty Git Repository

View Status of the Repository

$ git status
View the status of the repository

Add all files to git repository

$ git add .
Move all files of the directory to the staging area. Now, all your files are tracked by git. You can also use "git add --a" to stage/track all your files.

Add a specific file to git repository

$ git add {filename}
Move file to the staging area. Now, this file is tracked by git.

Commit changes to repository

$ git commit -m "Commit Message"
Commit the changes made to the repository. The state of the repository at the time of commit will be timestamped by git. Note that git captures snapshots of the repository, not differences between two commits. Also, try to set the commit message in such a way that it precisely describes the change made before commiting it.

Commit changes to repository skipping the staging area

$ git commit -a -m "Commit message"
Commit all the TRACKED and MODIFIED files directly, skipping the staging area.

See Commit History

$ git log
Check commit history with this command. It displays the username and email of the commit maker, along with the commit code, date and time, and also the commit message.

Deletions in Git Repository

Delete git repository

$ rm -rf .git
Delete the git repository by deleting the hidden .git directory which stores all the repository data. Note that this action does not delete your files, however, your git repository is destroyed.

Delete/Untrack specific file

$ git rm {filename}
Untrack file/ Delete file from git repository (Note that the file will not be deleted, however it will be untracked by git).
